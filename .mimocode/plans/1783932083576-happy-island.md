# Plan: Trip completion — auto-create end checkpoints + mobile warning

## Problem
Driver has orders completed (all checkpoints: Started → ArrivedPickup → LeftPickup → ArrivedDelivery → Completed) but **missing `end` checkpoint**. Trip completion modal shows orders as "Xong" but doesn't indicate missing end. When driver completes trip, no end checkpoint is created → checkpoint history incomplete.

## Root cause
1. Backend `Trip::complete()` does NOT auto-create `end` checkpoints for orders.
2. Mobile completion modal (`trip-detail.tsx:338-374`) only shows order status, not end checkpoint existence.

## Solution (chosen by user)
**Mobile cảnh báo + backend auto-create**

---

## Phase 1: Backend — auto-create end checkpoint on trip complete ✅ DONE

### File: `app/Services/Trip/TripCheckpointService.php`

Add method `createEndCheckpointsForCompletedOrders(Trip $trip, float $endKm, string $occurredAt): void`

Logic:
- For each order in the trip where status = Completed AND no `end` checkpoint exists:
  - Create a `TripCheckpoint` with:
    - `checkpoint_type` = `end`
    - `trip_id` = trip.id
    - `order_id` = order.id
    - `km_reading` = endKm
    - `occurred_at` = completedAt (or now)
    - `driver_id` = trip.driver_id
    - `shift_id` = trip.shift_id

### File: `app/Models/Trip.php`

Modify `complete()` method (line 139-160):
- After setting trip status + km, call `TripCheckpointService::createEndCheckpointsForCompletedOrders($this, $endKm, $completedAt)`
- Use DI: inject via app() or constructor

### File: `app/Http/Controllers/Api/TripController.php`

Modify `complete()` method (line 205-261):
- After `$trip->complete()` succeeds, load `checkpoints` relation to include newly created end checkpoints in response
- Already loads checkpoints at line 253-256, just ensure ordering

## Phase 3: Remove "same destination" restriction for auto-complete

### Problem
`CheckpointEndHandler::canAutoComplete()` requires all orders have the SAME last delivery point location_id. When orders have different destinations, the trip won't auto-complete even though all orders are done with end checkpoints.

### Fix: `app/Services/Trip/Handlers/CheckpointEndHandler.php`

Remove the `canAutoComplete()` method and its guard at line 48. The pending order check at line 40-46 already ensures only trips with all orders completed trigger auto-complete. No need for the destination check.

**Before (line 48-49):**
```php
if (! $this->canAutoComplete($orderIds)) {
    return;
}
```

**After:** Remove lines 48-50 and lines 55-77 (the entire `canAutoComplete` method).

Also remove unused `OrderDeliveryPoint` import and `Collection` import if not used elsewhere.
