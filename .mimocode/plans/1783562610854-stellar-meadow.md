# Plan: Group checkpoint display in Filament trip form

## Problem

When a trip has multiple orders at the same delivery location, the "Các mốc hành trình" section in the trip edit form (`TripForm.php` lines 105-148) shows one row per checkpoint. With 5 orders, each checkpoint type creates 5 identical rows (same type, same km, same time, different order_code). This results in ~35 rows of repetitive data that is hard to read.

## Solution

Replace the Repeater with a **custom View component** that groups checkpoints by `(checkpoint_type, km_reading)` and displays each group as a single compact row with all associated order codes combined.

### Approach

1. **Create a Blade view** at `resources/views/filament/resources/trips/components/grouped-checkpoints.blade.php`
   - Group checkpoints by `(checkpoint_type_value, km_reading)` using PHP collection methods
   - For each group, display one row with: type label, km, time, combined order codes
   - Use colored badges for checkpoint types (matching the timeline popup style)
   - Show order codes as inline tags/pills

2. **Modify `TripForm.php`** lines 105-148:
   - Replace the `Repeater` with `ViewField::make('grouped_checkpoints')` pointing to the new Blade view
   - Pass the trip's checkpoints as a view variable

3. **Also apply to `DriverShift` activity timeline** if feasible:
   - The shift timeline (`DriverShift::getActivityTimelineAttribute()`) also shows individual checkpoint rows
   - Group trip-wide checkpoints where multiple orders share the same checkpoint data

### Files to modify

| File | Change |
|------|--------|
| `resources/views/filament/resources/trips/components/grouped-checkpoints.blade.php` | **NEW** — Blade view with grouped checkpoint table |
| `app/Filament/Resources/Trips/Schemas/TripForm.php` | Replace Repeater (lines 105-148) with ViewField |
